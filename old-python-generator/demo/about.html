Backlinks

Backlinks are awesome for a few reasons:

  1. They allow you to build a site graph and see which are the most popular pages.
  2. They generalise the concept of "tag": a tag is simply a page with many backlinks and no content.
  3. They allow bidirectional traversal of the knowledge grpah.

It's a shame we don't see them very often. In general, wikis support them, for example here's the backlinks to 'KISS principle' wiki page. Personal Knowledge Graph software uses them a lot as well. Examples include Logseq, Athens, Obsidian and probably others.

HTML

Markdown might be simpler to write compared to HTML but it has a few downsides:

  1. Would pull in extra dependencies
  2. Requires an extra step to compile
  3. Would still require the use of HTML for things like custom-formatting

Python

A few reasons:

  1. Ubiquitous at this point.
